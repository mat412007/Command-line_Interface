Command-line Interface.

Guía de ejercicios.


  1. Indicar qué significan los acrónimos GUI, CLI, y TUI.

    2. GUI = Graphical User Interface.
    CLI = Command Line Interface.
    TUI = Text User Interface.


  2. ¿Cuál es la diferencia entre un shell y un emulador de terminal?

    3. La diferencia es que un shell es la interfaz con el sistema operativo, incorporan el lenguaje de programación para controlar procesos y archivos.
    Un emulador de terminal es una app de software que permiten usar ese lenguaje y mostrar resultados en una ventana local.


  3. Dar tres diferencias entre una GUI y una CLI.

    4. Los usuarios CLI escriben comandos manuales, los GUI usan visuales como botones e íconos. GUI se usa con teclado y mouse, requiere poca experiencia, es poco preciso, lento, y pesado, pero puede personalizarse.
    El CLI se usa solo con teclado, y requiere práctica y experiencia, pero es muy preciso, rápido, y ligero, pero no puede personalizarse.


  4. ¿Qué es el prompt en la CLI? ¿Cuáles son las partes del prompt típico de Bash?

    5. Un prompt es una interfaz de línea de comandos(CLI) que le permite al usuario interactuar con el sistema mediante comandos. Las partes del prompt típico en Bash son: username, hostname, directorio, separador, cursor.


  5. ¿En qué sentido el sistema de archivos de Windows y Linux son similares y en qué sentido son diferentes?

    6. Son similares en que ambos:

    • Manejan múltiples usuarios.
    • Ejecutan múltiples acciones.
    • Son compatibles con el manejo de redes.
    • Permiten el desarrollo de aplicaciones.
    • Requieren más recursos de hardware.
    • Utilizan ventanas, escritorios y papelera de reciclaje.
    • Pueden tener varias cuentas.

    Se diferencian en que:
    • Linux es de código abierto y cada usuario tiene sus archivos.
    • Windows es de código cerrado y fue creado por Microsoft para uso personal y comercial.


  6. ¿Qué es un archivo texto y en qué se diferencia de un archivo binario?

    7. Un archivo de texto es un registro de sistema en el que se almacenan datos de documentos de texto, y nada más, por eso sus nombres siempre terminan con TXT. Su diferencia con un archivo binario es que aunque ambos fueron escritos por el usuario, el binario es leído e interpretado por la computadora también. Le dicen al sistema qué hacer y en ellos se almacenan mandatos y programas.


  7. ¿Qué significa un punto delante del nombre de un archivo?

    8. Significa que el archivo está oculto y ls no lo mostrará, a menos que insertes ls -a.


  8. ¿Cuáles son las 4 clases de comandos que podemos encontrar en Linux?

    • Un programa ejecutable: dentro de esta categoría, los programas están escritos en lenguajes de script, y pueden ser binarios, escriots en C y C++.
    • Un comando contenido en el propio shell: bash soporta un gran número de comandos, llamados shell builtins.
    • Una función del shell: pequeños scripts de shell incorporados en el entorno.
    • Un Alias: comandos que definimos por nosotros mismos.

    Stderr = Standard Error.
